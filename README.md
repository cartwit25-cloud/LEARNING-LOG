# 學習手帳 — 設定與部署指南

這個工具包含兩個檔案：
- `index.html`：前端網頁（之後放到 GitHub Pages）
- `Code.gs`：後端程式（貼到 Google Apps Script）

沒有網路連線或還沒設定好 API 網址時，資料會先存在瀏覽器本機（localStorage），可以正常使用，只是不同裝置間不會同步。設定好之後，資料會自動同步到 Google Sheets。

---

## 步驟一：建立 Google 試算表

1. 到 [Google Sheets](https://sheets.new) 建立一份新的空白試算表，取名例如「學習手帳資料庫」
2. 從網址列複製試算表 ID（網址長這樣：`https://docs.google.com/spreadsheets/d/【這一段就是ID】/edit`）

## 步驟二：設定 Apps Script

1. 在這份試算表點選「擴充功能 → Apps Script」
2. 把預設的 `Code.gs` 內容全部刪除，貼上這個專案附的 `Code.gs` 內容
3. 找到程式最上面的這一行：
   ```js
   const SPREADSHEET_ID = '請貼上你的試算表ID';
   ```
   把 `請貼上你的試算表ID` 換成步驟一複製的 ID
4. 在上方函式選單選擇 `setup`，按執行（▶️）
   - 第一次執行會跳出授權視窗，選你的 Google 帳號 → 「進階」→「前往（專案名稱）」→ 允許
   - 執行完成後回到試算表，應該會看到自動建立了 `Notes`、`Categories`、`Relations`、`Positions` 四個工作表

## 步驟三：部署成網頁應用程式

1. 在 Apps Script 編輯器右上角點「部署 → 新增部署作業」
2. 類型選「網頁應用程式」
3. 設定：
   - 說明：學習手帳 API（隨意）
   - 執行身分：**我**
   - 存取權限：**任何人**
4. 點「部署」，會出現一個網址，長得像：
   `https://script.google.com/macros/s/xxxxxxxx/exec`
5. 複製這個網址

> 之後如果修改了 `Code.gs`，記得要「管理部署作業 → 編輯 → 新版本」重新部署，網址才會套用新程式碼。

## 步驟四：設定前端

打開 `index.html`，找到：
```js
const API_URL = '請貼上你的 Google Apps Script 網頁應用程式網址';
```
換成步驟三拿到的網址。

## 步驟五：部署到 GitHub Pages

1. 建一個新的 GitHub repository（可以設成 public 或 private，Pages 免費版建議 public）
2. 把改好的 `index.html` 上傳上去
3. 到 repo 的 Settings → Pages，Source 選擇你放檔案的分支（通常是 `main`），資料夾選 `/root`
4. 存檔後幾分鐘，就會拿到一個 `https://你的帳號.github.io/repo名稱/` 的網址，手機電腦都能打開使用

## 功能說明

- **列表 / 卡片 / 心智圖（自動）/ 心智圖（自由編輯）**：上方分頁可以切換，資料共用同一份，不用重複輸入
- **心智圖・自動**：依「分類」自動長出放射狀心智圖，點分類節點可以展開／收合，點筆記節點可以打開該筆記
- **心智圖・自由編輯**：所有筆記都是可以拖曳的節點，開啟「連線模式」後依序點兩個節點就能建立自訂關聯（例如「這兩篇筆記其實在講同一件事」），排列位置與連線都會存回 Google Sheets
- **複習提醒**：首頁上方會自動列出「6～9 天前」記錄、還沒標記已複習的筆記，點✓即可標記
- **附圖**：新增筆記時可以上傳圖片（例如書頁截圖），會自動壓縮後存到你 Google Drive 裡一個叫「學習筆記圖片」的資料夾
- **匯出 PDF**：兩種心智圖畫面都有「匯出 PDF」按鈕

## 目前的簡化設計（之後想要更完整可以再加強）

1. **大量筆記的效能**：目前心智圖用 SVG 直接渲染所有節點，筆記在一兩百篇以內都會很流暢。如果之後累積到數百篇以上，建議：
   - 自動心智圖：預設把節點收合、只展開常用分類
   - 自由編輯心智圖：可以加上「只顯示某個分類」的篩選，減少同時渲染的節點數
   - 更進一步可以改用 Canvas 或 WebGL 渲染（例如 pixi.js），但目前規模不需要
2. **多裝置同步**：目前是「本機優先、背景同步」的策略（先寫入畫面與 localStorage，再送到 Google Sheets），單人使用不會有衝突問題；但如果你同時在手機和電腦快速新增筆記，極少數情況可能會有寫入順序的小誤差，重新整理一次即可。
3. **圖片儲存**：圖片是存在你自己的 Google Drive，Sheets 裡只存網址；如果要澈底離線使用，可以之後再加上圖片快取機制。
4. **分類沒有層級**：目前分類是扁平的（不分主分類、子分類），如果之後想要「分類底下還有子分類」，可以再擴充 `Categories` 工作表的欄位。

## 疑難排解

- **畫面資料一直沒有更新 / 心智圖是空的**：先檢查 `index.html` 裡的 `API_URL` 是否已經換成你自己的部署網址，且該網址結尾是 `/exec`（不是 `/dev`）
- **新增筆記後，重新整理就不見了**：代表 Apps Script 沒有部署成功，或存取權限沒選「任何人」，回到步驟三重新檢查
- **圖片上傳失敗**：確認 `setup()` 執行時有完整授權 Google Drive 的權限（第一次授權若中途取消，需要重新執行一次 `setup`）
